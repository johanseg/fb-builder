1. Issues = The Assignment
An Issue is a dedicated ticket for a specific task. Instead of a messy email or a Slack message that gets lost, an Issue acts as the "home" for a piece of work.

What goes in it: The creative brief, the goal, the deadline, and the discussion.

Marketing Example: You create an Issue titled "Write Q3 Newsletter." All brainstorming and feedback happen inside that ticket so there is a permanent record of the decisions made.

2. Pull Requests (PRs) = The Review Room
A Pull Request is how you submit finished work for approval. It literally means "I have finished this content; I am requesting that you pull it into our main brand folder."

What goes in it: The actual draft (copy, code, or images) compared against the previous version.

The Power of PRs: It allows a manager (or Claude) to see exactly what changed. They can leave comments on specific lines of text, ask for edits, and finally "Merge" it (mark it as officially finished/published).
